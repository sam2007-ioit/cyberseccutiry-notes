principles of Open Worlwide Application Security Project (OWASP):
  1. Reduce surface area of attack : to reduce the vulnerabilities that can be exploited by threat actor
  2. Least privilage principle : it means that not to give too much access to someone if in any case the system of the person with all access got compromised then the threat actor will get alot of information
  3. Defence in depth : to add multiple layers of defence which will make it hard for the threat actor to break through
  4. Distribute the responsibility : one person can't keep watch on multiple things at one time so the responsibility should be distributed so that the defence will be at its peak
  5. Keep security simple : don't make the security so complicated that it will make you difficult for you to control if any data breach occured
  6. Fix security issues properly : to properly investigate how actually the breach occcured, which system or data were compromised, so that next time these vulnerabilities will be fixed
  7. fail securly : imagine a condition where your phone got lost now if you have activated the fake power off, passwards, live location access then it will be easy to track back and due to passwards the information inside will also be safe
  8. secure defaults : consider you are changing the controls of a game to make it more confortable to play but while changing it you did something wrong now all the controls are different and hard, you press the "change to default button" and you controls go back to default again in same way security should also have some secure default
  9. don't trust services : never to give complete access to some third party service your data can also get compromised by them
